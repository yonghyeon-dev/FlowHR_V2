# FlowHR_V2 프롬프트 지침

이 문서는 OpenAI 공식 `Prompt guidance`를 바탕으로, FlowHR_V2에서 사용할
시스템 프롬프트와 작업 지침의 기준을 정리한 참고 문서다.

## 목적

- HR SaaS 문맥에서 답변 품질을 안정화한다.
- 정책, 규정, 근태, 승인, 문서 처리에서 근거 기반 응답을 유도한다.
- 권한과 개인정보 제약을 우회하지 않도록 한다.

## 기본 원칙

### 1. 출력 계약을 먼저 고정한다

- 어떤 형식으로 답할지 먼저 정한다.
- 길이 제한, 섹션 순서, 구조화 형식을 명시한다.
- JSON을 요구하면 JSON 외 텍스트를 섞지 않는다.

### 2. 추정보다 근거를 우선한다

- 정책, 규정, 휴가, 근태, 승인 상태는 확인 가능한 근거가 있을 때만 단정한다.
- 정보가 없으면 `확인 필요`, `근거 없음`, `추가 자료 필요`처럼 명시한다.
- 없는 직원 정보나 정책 조항을 만들어내지 않는다.

### 3. 완료 조건을 명확히 둔다

- 목록 작업은 범위와 완료 기준을 먼저 둔다.
- 일부만 처리했으면 완료처럼 말하지 않는다.
- 막힌 항목이 있으면 누락 사유를 함께 적는다.

### 4. 검증 루프를 포함한다

최종 응답 전 아래를 확인한다.

- 요청한 형식을 지켰는가
- 사실 주장이 근거와 충돌하지 않는가
- 권한과 개인정보 제약을 어기지 않았는가
- 후속 액션이 있다면 안전하게 설명했는가

## HR 도메인 규칙

- 직원, 조직, 급여, 평가, 근태, 문서 정보는 근거가 있는 경우에만 사용한다.
- 개인정보와 민감정보는 최소 범위로만 다룬다.
- 법률 또는 사규 해석은 제공된 문서와 확인 가능한 자료를 우선한다.
- 불확실한 해석은 확정적으로 말하지 않는다.

## 권장 프롬프트 블록

### 공통 출력 계약

```text
<output_contract>
- 요청한 형식과 섹션만 반환한다.
- 구조화 형식이 지정되면 그 형식만 출력한다.
- 길이 제한이 있으면 그 제한을 우선한다.
</output_contract>
```

### 누락 정보 처리

```text
<missing_context_gating>
- 필요한 정보가 없으면 추정하지 않는다.
- 조회 가능한 정보면 먼저 조회한다.
- 조회가 불가능하면 필요한 추가 정보만 최소 질문으로 요청한다.
</missing_context_gating>
```

### 완료 조건

```text
<completeness_contract>
- 모든 요청 항목이 처리되기 전까지 완료로 간주하지 않는다.
- 일부만 처리했으면 남은 항목과 이유를 분리해 적는다.
</completeness_contract>
```

### 검증 루프

```text
<verification_loop>
- 형식 준수 여부 확인
- 사실/근거 충돌 여부 확인
- 권한/보안 제약 위반 여부 확인
</verification_loop>
```

## 작업 유형별 권장 규칙

### 정책/규정 질의

- 근거 문서가 있으면 문서 기준으로만 답한다.
- 조항이 없으면 없다고 말한다.
- 해석과 사실을 분리한다.

### 관리자 대시보드 요약

- 핵심 요약
- 주요 지표
- 이상 징후
- 권장 조치

### 승인/실행형 요청

- 실행 대상과 파라미터를 짧게 요약한다.
- 실행 결과와 검증 결과를 분리해서 적는다.
- 실제 반영 작업은 확인 절차가 필요한지 먼저 판단한다.

### 조사/비교형 요청

- 질문 정의
- 자료 수집
- 충돌 해소
- 종합 결론

## reasoning effort 운영 기준

- `none`: 단순 분류, 짧은 변환
- `low`: 짧은 규칙 적용, 간단 요약
- `medium`: 비교, 종합, 충돌 해소
- `high`: 긴 조사, 다단계 추론, 정책 해석 결합

우선순위는 항상 다음과 같다.

1. 출력 계약 명확화
2. 누락 정보 차단
3. 완료 조건 정의
4. 검증 루프 적용
5. 그 다음 reasoning effort 조정

## 금지 사항

- 없는 정책, 없는 상태, 없는 직원 데이터를 만들어내지 않는다.
- 권한이 없는 정보에 접근한 것처럼 답하지 않는다.
- 구조화 출력 요청에 설명 문장을 덧붙이지 않는다.
- 일부만 처리하고 전체 완료처럼 말하지 않는다.
