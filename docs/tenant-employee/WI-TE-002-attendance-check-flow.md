# WI-TE-002 출퇴근 플로우 상세 설계

## 작업 ID

- `WI-TE-002`

## 목적

직원 출퇴근 경험은 단순 버튼 탭이 아니라 `현재 상태`, `위치/정책 조건`,
`예외 발생 가능성`, `정정 진입`을 함께 다루는 플로우여야 한다.

## 대상 화면

- `TE-102` Attendance Check In/Out
- `TE-103` My Attendance History

## 핵심 UX

- 현재 상태를 첫 카드에서 명확히 보여준다.
- 출근, 퇴근, 외근, 휴게가 같은 액션 그룹 안에 있어야 한다.
- 예외가 발생하면 즉시 이유와 정정 진입점을 보여준다.
- 기록 이력은 조회용이 아니라 수정 요청 진입 허브여야 한다.

## 상태 정의

- 출근 전
- 근무 중
- 휴게 중
- 외근 중
- 퇴근 완료
- 예외 발생

## 예외 연결

- 체크인 누락
- 체크아웃 누락
- 지각
- 조퇴
- 휴게 위반
- 위치 불일치

## 주요 액션

- 출근
- 퇴근
- 외근 시작 / 종료
- 휴게 시작 / 종료
- 정정 요청
- 기록 확인
