# FlowHR_V2 단계 게이트

## 목적

단계 진행 상태를 임의 해석이 아니라 고정된 기준으로 판정하기 위한 문서다.
이 문서는 이후 설계 루프와 상태 보고의 최상위 기준점이다.

## 운영 원칙

- 단계 판단은 이 문서 기준으로만 내린다.
- 일부 문서가 추가되었다고 상위 단계를 완료로 올리지 않는다.
- 다음 단계로 넘어가려면 현재 단계의 필수 산출물이 모두 충족되어야 한다.
- 이후 보고는 항상 `현재 단계`, `충족`, `미충족`, `다음 작업` 순서로 정리한다.

## 현재 공식 상태

- 현재 단계: `Stage 4, Stage 6 진행 중`
- 정확한 표현: `상세 설계 진행 단계`
- 금지 표현:
  - `구현 직전`
  - `설계 완료`
  - `바로 구현 가능`

## 현재 판정 근거

- Stage 4:
  - 코어 와이어와 WI 문서는 존재한다
  - 일부 코어 화면은 승인 로그까지 연결되었다
  - 다만 사용자 최종 decision이 아직 `approved`로 확정되지 않았다
- Stage 6:
  - 라우트, 상태관리, API, i18n 구조는 정리되었다
  - 기술 선택안과 화면별 구현 연결 세분화가 계속 진행 중이다
- Stage 7:
  - 승인 로그가 아직 `hold` 중심이고, Stage 4와 Stage 6이 완료 판정에 도달하지 않았다

## 다음 작업 순서

3. 사용자 decision 반영을 위한 승인 패스 유지
4. `approved` 화면 R5 승격
