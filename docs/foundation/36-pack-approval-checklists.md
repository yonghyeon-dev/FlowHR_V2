# FlowHR_V2 Pack 승인 체크리스트

## 목적

사용자 검토 시 바로 읽고 결정할 수 있도록 Pack별 질문을 짧게 정리한 문서다.

## Office Pack

### 관리자

- 승인/문서/휴가/근태 예외가 하나의 운영 흐름으로 읽히는가
- 첫 화면에서 가장 먼저 처리할 업무가 명확한가
- 유연근무, 재택, 정정 요청 같은 오피스형 예외가 자연스러운가

### 직원

- 요청 제출, 서명 확인, 일정 확인이 자연스럽게 이어지는가
- 직원 홈에서 불필요한 운영 정보가 노출되지 않는가

## Retail Pack

### 관리자

- 결원, 커버리지, 브레이크, 오버타임, 정산 전 예외가 우선순위대로 보이는가
- 매장 운영자가 당장 조치할 항목이 첫 화면에 올라오는가

### 직원

- 오늘 근무, 체크인, 시프트 변경 응답, 필수 서명 확인이 자연스러운가
- 근무 시작 전 꼭 필요한 정보만 보이는가

## Platform

- 운영 지표가 모두 의미 있는가
- 각 지표가 다음 액션과 연결되는가
- 장식 요소 없이 리스크 우선순위가 보이는가

## 결정 형식

- `approved`
- `changes_requested: 사유`
- `hold`
