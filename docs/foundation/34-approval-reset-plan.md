# FlowHR_V2 승인 리셋 계획

## 목적

이 문서는 기존 승인 후보 화면들을 그대로 밀지 않고,
업종 기준이 고정되기 전까지 승인 프로세스를 재정렬하기 위한 계획 문서다.

## 리셋이 필요한 이유

현재 승인 후보 세트는 아래 문제를 안고 있다.

- 업종 기준이 고정되기 전에 생성됐다
- `권한별 화면`은 정리됐지만 `업종별 의사결정` 기준이 약하다
- 일부 화면은 운영 액션보다 범용 대시보드 구조가 먼저 보인다
- 최소 한 개 이상의 화면에서 의미 없는 시각 요소가 확인됐다

따라서 지금 필요한 것은 `승인 속도`가 아니라 `승인 기준 재정의`다.

## 현재 승인 상태 해석

- 기존 `APP-SCREEN-*` 로그는 폐기하지 않는다
- 다만 현재 상태는 `품질 통과 대기`가 아니라 `재검토 대기`로 해석한다
- 특히 1차 구현 진입 후보 세트는 업종 기준 문서가 생기기 전까지 `hold` 유지가 맞다

## 새 승인 기준

앞으로 승인 후보는 아래 6개를 먼저 통과해야 한다.

### 1. 업종 명시

- 이 화면이 어느 업종 팩 기준인지 명시돼야 한다
- `공통 화면`이면 왜 공통으로 유지 가능한지 근거가 있어야 한다

### 2. 역할 명시

- 어떤 사용자가 어떤 시점에 쓰는지 분명해야 한다

### 3. 결정 명시

- 이 화면에서 사용자가 무엇을 결정하는지 한 문장으로 설명 가능해야 한다

### 4. 액션 명시

- 화면 안의 핵심 카드나 지표는 반드시 다음 행동과 연결돼야 한다

### 5. 업종 반증 검토

- 어느 업종에서 바로 깨지는지 먼저 적어야 한다
- 깨진다면 공통 화면으로 두지 않는다

### 6. 시각 장식 배제

- 라벨 없는 그래프
- 좋은지 나쁜지 모르는 KPI
- 정보는 있으나 행동으로 이어지지 않는 카드

이 세 가지가 있으면 승인 후보로 올리지 않는다

## 승인 프로세스 변경

### 기존

- 화면 생성
- 승인 패킷 등록
- 사용자 검토

### 변경

- 업종 팩 지정
- 역할/결정 시나리오 연결
- 비관적 검토 루프 통과
- 승인 패킷 등록
- 사용자 검토

## 비관적 검토 루프

각 승인 후보는 아래 질문에 답해야 한다.

1. 이 화면이 왜 실무에서 버려질 수 있는가
2. 이 화면은 어떤 업종에서 우선순위가 틀어지는가
3. 이 화면에 장식성 요소가 있는가
4. 이 화면에서 가장 중요한 액션은 무엇인가
5. 더 적은 구역으로 더 분명하게 표현할 수 있는가

## 바로 이어질 작업

### 1. `TA-001`을 업종별 홈으로 재분해

- Office Home
- Retail Home
- Manufacturing Home은 후순위 초안

### 2. `TE-001`, `TE-002`를 업종별 첫 행동 기준으로 재구성

- Office:
  - 요청
  - 서명
  - 휴가
- Retail:
  - 오늘 근무
  - 체크인
  - 변경 확인

### 3. `PC-001`을 운영 리스크 패널 중심으로 재설계

- 모든 위젯에 지표명
- 현재값
- 상태
- 임계치
- 액션 연결

## 현재 권장 결론

지금은 `approved`를 늘릴 단계가 아니라,
`승인 후보를 다시 자를 단계`다.

따라서 다음 순서는 아래가 맞다.

1. 업종 세분화 확정
2. 업종 팩 구조 확정
3. 역할별 의사결정 시나리오 확정
4. 승인 후보 재구성
5. 홈 화면 재설계
