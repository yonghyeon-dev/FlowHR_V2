# FlowHR_V2 역할별 의사결정 시나리오 맵

## 목적

이 문서는 기존의 `권한 -> 화면` 중심 구조를 보완하기 위해,
`결정 -> 역할 -> 업종 -> 화면` 기준으로 제품을 다시 정렬하기 위한 문서다.

핵심 질문은 하나다.

- `이 사용자는 오늘 무엇을 결정해야 하는가`

## 공통 원칙

- 화면은 정보를 예쁘게 나열하는 용도가 아니라 결정 속도를 올리는 용도여야 한다
- 같은 역할이라도 업종이 다르면 첫 화면과 우선 액션이 달라져야 한다
- 홈 화면은 `요약`이 아니라 `의사결정 대기열`이어야 한다

## 홈 화면 재설계에 주는 시사점

### 1. 홈 화면은 업종별로 나뉘어야 한다

- `TA-001` 하나로 Office, Retail, Manufacturing을 모두 만족시키기 어렵다
- 최소 `Office Home`, `Retail Home`은 분리 설계가 필요하다

### 2. 직원 홈도 업종별 진입점이 달라야 한다

- Office는 `요청/서명/휴가`
- Retail과 Manufacturing은 `오늘 근무/체크인/변경 확인`

### 3. 플랫폼 콘솔은 장식형 그래프를 배제해야 한다

- Platform은 업종 차이보다 `운영 리스크`가 중요하다
- 따라서 모든 위젯은 `지표명 + 상태 + 임계치 + 액션`이 있어야 한다
