# WI-DM-001 근태 예외 유형 정의

## 작업 ID

- `WI-DM-001`

## 목적

근태 예외는 UI 배지 수준의 태그가 아니라, 관리자 운영과 직원 셀프서비스를 연결하는
공통 도메인 규칙이다. 이 문서는 예외 유형, 탐지 조건, 사용자 노출 방식의 기준을 정한다.

## 예외 분류 체계

### A. 기록 누락

- 체크인 누락
- 체크아웃 누락
- 휴게 시작/종료 누락

### B. 시간 위반

- 지각
- 조퇴
- 무단 결근 추정
- 휴게 미준수

### C. 정책 위반

- 주간 근무 한도 임박
- 초과근무 한도 초과
- 연속 야간근무
- 승인 없는 연장근무

### D. 맥락 충돌

- 위치 불일치
- 스케줄 대비 다른 근무 상태
- 휴가와 출근 기록 충돌
- 중복 기록

## 탐지 조건 원칙

- 정책과 조직 설정을 우선 기준으로 삼는다.
- 단순 시간 차이와 확정 위반을 구분한다.
- 추정 상태와 확정 상태를 같은 심각도로 보여주지 않는다.

## 심각도 정의

- `critical`: 법규/정산/보안 리스크가 큰 경우
- `high`: 당일 운영에 즉시 영향이 있는 경우
- `medium`: 확인과 정정이 필요한 경우
- `low`: 참고 알림 수준

## 사용자 노출 방식

### Tenant Admin

- 예외 큐
- 대시보드 경고
- 마감 전 검토 리스트

### Tenant Employee

- 출퇴근 플로우 내 즉시 경고
- 요청 허브의 정정 진입
- 내 기록 이력의 상태 배지

## 상태 흐름

- 감지됨
- 확인 필요
- 정정 요청됨
- 승인 대기
- 해소됨
- 반려됨

- `WI-TA-003` Attendance 모듈
- `WI-TE-002` 출퇴근 플로우
- `WI-TE-003` 요청 허브
