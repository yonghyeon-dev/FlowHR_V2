# WI-TA-008 Performance 모듈 상세 설계

## 작업 ID

- `WI-TA-008`

## 목적

Performance 모듈은 평가 시즌에만 쓰는 화면이 아니라, 목표, 평가 사이클,
1:1, 피드백을 계속 이어서 관리하는 성장 운영 모듈이어야 한다.

## 화면 구성

- `TA-701` Goal Dashboard
- `TA-702` Evaluation Setup
- `TA-703` Evaluation Cycle
- `TA-704` One-on-One Hub

## 핵심 UX

- 전체 평가 일정과 병목 상태를 먼저 보여준다.
- 목표 진행률과 평가 상태를 분리하되 연결해서 읽히게 한다.
- 1:1과 피드백은 HR 관리 화면처럼 무겁지 않게 다뤄야 한다.
- 조직장과 HR이 같은 화면을 보더라도 관점이 달라야 한다.

## 운영 포인트

- 평가 주기 설정
- 템플릿 / 문항 관리
- 자기평가 / 상향 / 동료 / 종합 평가
- 1:1 일정과 메모
- 누락 평가자 추적

## 주요 액션

- 평가 사이클 생성
- 문항 수정
- 진행 상태 점검
- 리마인더 발송
- 결과 검토
