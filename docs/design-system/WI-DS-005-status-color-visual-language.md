# WI-DS-005 상태 색상과 시각 언어 규칙

## 작업 ID

- `WI-DS-005`

## 목적

FlowHR_V2는 근태 예외, 승인 병목, 문서 미서명, 결제 실패, 보안 경고처럼
상태 기반 판단이 많은 HR SaaS다. 따라서 색상은 장식이 아니라 상태 해석과
우선순위 판단을 돕는 운영 언어여야 한다.

## 시각 언어 원칙

- 기본 정보는 대비와 위계로 읽히게 하고, 색상은 상태 차이를 보강하는 용도로 사용한다.
- 같은 상태 의미는 권한 레이어가 달라도 같은 시각 언어를 유지한다.
- 위험도는 `critical > high > medium > neutral > success` 순으로 인지되게 설계한다.
- 관리자 화면은 밀도 높은 정보 안에서도 상태가 먼저 읽혀야 하고, 직원 화면은 불안감을 과도하게 유발하지 않도록 절제한다.
- 텍스트, 아이콘, 배경 tint, 배지, 표 행 강조가 한 세트로 동작해야 한다.

## 의미 토큰

### 1. Critical

- 의미: 즉시 조치가 필요한 보안/운영 위험
- 사용 예:
  - 결제 실패로 인한 서비스 제한 임박
  - 운영자 민감 권한 승격
  - 급여 마감 차단 이슈
  - 전자서명 마감 초과
- 표현 규칙:
  - 진한 경고 배경 tint
  - 강한 텍스트 대비
  - 큐 카드나 테이블 행 상단에 우선 노출

### 2. High

- 의미: 당일 내 조정이 필요한 예외 또는 병목
- 사용 예:
  - 체크아웃 누락 다수
  - 승인 적체
  - 특정 부서 초과근무 임박
- 표현 규칙:
  - 경고 계열 tint
  - 강조 배지
  - 필터 기본값에서 우선 정렬 가능

### 3. Medium

- 의미: 추적과 확인이 필요한 상태
- 사용 예:
  - 정책 검토 필요
  - 서명 대기
  - 예정된 마감 작업
- 표현 규칙:
  - 브랜드 accent tint
  - 정보형 카드와 구분되는 중간 톤

### 4. Neutral

- 의미: 일반 정보, 참조 상태, 히스토리
- 사용 예:
  - 기본 프로필 정보
  - 완료 로그
  - 일반 리포트
- 표현 규칙:
  - 저채도 배경
  - 라인과 타이포그래피 중심

### 5. Success

- 의미: 정상 완료, 적용 완료, 정책 충족
- 사용 예:
  - 급여 마감 완료
  - 서명 완료
  - MFA 적용 완료
- 표현 규칙:
  - 초록 계열 tint
  - 완료 상태 배지
  - 경고 계열과 경쟁하지 않도록 면적은 작게 사용

## 권한 레이어별 적용 기준

### Platform Operator

- `critical`: 보안 이상, 서비스 제한, 결제 실패, 권한 오남용
- `high`: 지원 SLA 임박, 감사 검토 대기
- `medium`: 예정된 정책 변경, 모니터링 주시

### Tenant Admin

- `critical`: 급여 마감 차단, 결재 병목, 법정 근로 위반 가능성
- `high`: 근태 누락, 서명 지연, 승인 적체
- `medium`: 예정된 인사 변경, 정책 공지, 리포트 이상 징후

### Tenant Employee

- `critical`: 거의 사용하지 않는다. 직원 화면에서는 강한 위험 표현을 최소화한다.
- `high`: 서명 마감 임박, 요청 반려, 출퇴근 정정 필요
- `medium`: 일정 알림, 잔여 연차 안내, 프로필 보완 요청

## 컴포넌트 적용 규칙

### 배지

- 짧은 상태 라벨에만 사용한다.
- 같은 카드 안에서 배지 수는 최대 2개로 제한한다.
- 배지 텍스트는 번역 키로 분리한다.

### 큐 카드

- 카드 전체 tint는 상태 1개만 대표로 사용한다.
- 보조 정보는 색상보다 구조와 텍스트로 구분한다.

### 테이블

- 처리형 테이블은 행 강조를 허용한다.
- 조회형 테이블은 배지와 아이콘 중심으로 절제한다.

### 모바일

- 전체 카드 배경을 과도하게 칠하지 않는다.
- 핵심 배지와 CTA만 상태색을 쓴다.

## 접근성 기준

- 상태 의미는 색상만으로 전달하지 않는다.
- 배지, 아이콘, 텍스트 라벨을 함께 사용한다.
- 주요 상태 텍스트 대비는 최소 AA 수준을 목표로 한다.
- 위험 상태는 hover나 툴팁 없이도 의미가 읽혀야 한다.

## 로컬라이제이션 연계

- 상태 라벨은 모두 `LID-*`로 관리한다.
- 예:
  - `LID-COMMON-STATUS-CRITICAL`
  - `LID-COMMON-STATUS-HIGH`
  - `LID-COMMON-STATUS-MEDIUM`
  - `LID-COMMON-STATUS-NEUTRAL`
  - `LID-COMMON-STATUS-SUCCESS`
