# WI-DS-001 디자인 시스템 방향

## 목적

FlowHR_V2의 와이어프레임을 실제 제품 UI로 발전시키기 위한 시각 방향을 정의한다.

## 방향

- 업무용 SaaS 특유의 밀도는 유지한다.
- 정보가 많은 화면에서도 위계가 분명해야 한다.
- 관리자 제품과 직원 제품의 톤은 같되 사용 감도는 달라야 한다.

## 원칙

- 관리자 화면: 밀도 높음, 명확한 구분, 강한 상태 표현
- 직원 화면: 단순한 흐름, 빠른 행동, 모바일 우선 감각
- 플랫폼 화면: 백오피스 톤, 리스크와 상태 중심

## 후속 항목

- 상태 색상 체계
- 표와 카드 혼합 규칙
- 모바일 탭 구조
