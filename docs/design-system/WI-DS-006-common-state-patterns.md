# WI-DS-006 공통 상태 및 예외 UX 패턴

## 작업 ID

- `WI-DS-006`

## 목적

FlowHR_V2는 정보 밀도가 높은 운영형 HR SaaS이기 때문에, 정상 상태 화면만 설계하면
구현 단계에서 UX 품질이 급격히 흔들린다. 이 문서는 모든 권한 레이어에서 공통으로
필요한 `loading`, `empty`, `error`, `no-permission`, `first-run`, `confirm`
패턴의 기준을 정의한다.

## 공통 원칙

- 상태 메시지는 시스템 내부 표현이 아니라 사용자가 다음 행동을 이해하게 해야 한다.
- 같은 상태 의미는 관리자/직원/플랫폼 화면에서 일관된 구조를 유지한다.
- 색상만으로 상태를 구분하지 않는다.
- 에러는 가능한 한 복구 액션을 함께 제시한다.

## 1. Loading

### 적용 대상

- 대시보드 KPI
- 테이블
- 상세 패널
- 문서 뷰어
- 리포트 추출

### 원칙

- 전체 페이지를 막는 loading은 최초 진입이나 파괴적 전환에만 사용한다.
- 카드와 테이블은 skeleton 기반 loading을 기본으로 한다.
- 문서/리포트처럼 시간이 긴 작업은 진행 상태와 예상 소요를 함께 준다.

## 2. Empty

### 유형

- 아직 데이터가 없음
- 필터 결과가 없음
- 권한 범위 안에 표시할 항목이 없음
- 기능이 아직 활성화되지 않음

### 원칙

- `없음`의 이유를 분리해서 표현한다.
- 가능한 경우 첫 행동 CTA를 함께 둔다.
- 필터 결과가 없는 경우는 필터 리셋 경로를 먼저 제시한다.

예:

- 휴가 요청 없음: `첫 휴가 요청 만들기`
- 필터 결과 없음: `필터 초기화`
- 기능 미활성: `관리자에게 요청`

## 3. Error

### 유형

- 네트워크 실패
- 권한 검증 실패
- 서버 처리 실패
- 외부 연동 실패

### 원칙

- 사용자 책임과 시스템 책임을 구분한다.
- 재시도 가능한 에러는 `다시 시도` 액션을 기본 제공한다.
- 데이터 저장 실패는 손실 위험을 먼저 알려야 한다.
- 관리자 화면은 실패 원인 힌트를 더 많이 보여주고, 직원 화면은 불필요한 기술 용어를 줄인다.

## 4. No Permission

### 원칙

- 단순 빈 화면이 아니라 접근 불가 상태임을 분명히 알려야 한다.
- 메뉴 숨김과 별개로 URL 직접 진입 시도에 대한 상태 화면이 필요하다.
- 필요한 경우 권한 요청 경로를 제시한다.

예:

- Tenant Employee가 급여 관리 URL 접근
- Manager가 Payroll Closing 진입
- Platform Operator가 승인 없는 tenant 원문 데이터 접근

## 5. First-run / Setup

### 적용 대상

- 회사 초기 설정
- 근무 정책 템플릿
- 권한 세팅
- SSO / 연동
- 기능 플래그 베타 활성화

### 원칙

- 설정 순서는 실제 제품 의존 순서와 맞아야 한다.
- 건너뛰면 나중에 어떤 영향이 있는지 알려야 한다.
- Platform과 Tenant의 first-run UX는 혼동되지 않게 분리한다.

## 6. Confirm / Destructive Action

### 적용 대상

- 급여 마감
- 테넌트 중지
- 권한 회수
- 문서 일괄 발송
- 기능 플래그 비활성화

### 원칙

- 영향 범위, 되돌릴 수 있는지 여부, 대상 수를 먼저 보여준다.
- 파괴적 액션은 기본 버튼이 되면 안 된다.
- 민감 액션은 확인 문구 입력 또는 재인증을 고려한다.

## 7. Success / Toast / Feedback

### 원칙

- 저장 완료, 발송 완료, 승인 완료는 짧고 명확하게 보여준다.
- 다음 행동이 중요하면 토스트보다 inline feedback이나 summary panel을 우선한다.
- 대량 처리 작업은 `완료 수 / 실패 수 / 재시도` 구조로 보여준다.

## 권한 레이어별 차이

### Platform

- 위험도와 감사 추적 강조
- 민감 액션 경고 강도 높음

### Tenant Admin

- 운영 영향 범위와 마감 영향 강조
- 병목 처리 CTA 중요

### Tenant Employee

- 짧고 빠른 복구 경로 중요
- 심리적 부담을 줄이는 톤 유지
